The TPM Launchpad System

Proposed

Three parallel tracks every week. Each track produces real output — not just "learned" but done.

Code Track

3-4 hrs/week

Genuine programming proficiency. Python, git, APIs, databases, testing. Not "learn to code" — learn to read, review, estimate, and build.

Outputs: Working code, passing tests, PRs reviewed, CI pipelines

AI/Agent Track

1-2 hrs/week

Deep technical understanding of AI systems. Tokenization, transformers, function calling, agent loops, MCP, multi-agent systems. Not vibe coding.

Outputs: Working agents, MCP servers, tool definitions, eval suites

TPM Track

2 hrs/week

The actual TPM craft. Roadmaps, stakeholder management, technical specs, risk registers, architecture reviews, post-mortems.

Outputs: Tech specs, ADRs, program charters, stakeholder decks, risk registers

What Makes It Interactive

1.

pytest-Verified Exercises

Every code exercise has tests. Write code, run pytest, get pass/fail immediately.

2.

Anki Spaced Repetition

Pre-built Anki deck per phase. 10 min/day reviewing old material locks it in long-term.

3.

CI-Verified Progress

GitHub Actions runs tests on every push. Green checkmark = you're on track. Red = review needed.

4.

Progressive Capstone

One project that grows every week from a Python script to a full multi-agent system with frontend.

5.

Answer Keys & Rubrics

Every checkpoint has a reference answer. Every TPM artifact has a grading rubric for self-evaluation.

6.

Progress Dashboard

A static site (or CLI) showing weeks completed, tests passed, artifacts produced. Motivation + accountability.
